# 35_문항 QA 및 16유형 검증 리포트

## 1. 목적

본 문서는 로컬 MVP의 문항 응답 흐름이 16개 기본 유형을 모두 산출할 수 있는지, 그리고 MVP 문항/결과 카피가 최소한의 서비스 노출 품질을 만족하는지 검증하기 위한 기준과 현재 구현 상태를 정리합니다.

## 2. 검증 대상

- 서버 전용 문항 풀: `src/lib/server/data/questions.ko-KR.ts`
- 서버 전용 결과 매핑: `src/lib/server/data/result-mappings.ko-KR.ts`
- 문항 선택 알고리즘: `src/lib/server/algorithm/questions.ts`
- 점수 및 S/M 산출 알고리즘: `src/lib/server/algorithm/scoring.ts`
- 결과 ViewModel 조립: `src/lib/server/algorithm/results.ts`

## 3. 16유형 도달성 기준

16개 기본 유형은 `C/I`, `T/P`, `M/E`, `O/L` 네 축의 조합으로 산출됩니다.

| 축 | 왼쪽 코드 | 오른쪽 코드 |
|---|---:|---:|
| 공동체/개인 | C | I |
| 전통/진보 | T | P |
| 성과/평등 | M | E |
| 질서/자유 | O | L |

검증 기준:

1. `CTMO`, `CTML`, `CTEO`, `CTEL`, `CPMO`, `CPML`, `CPEO`, `CPEL`, `ITMO`, `ITML`, `ITEO`, `ITEL`, `IPMO`, `IPML`, `IPEO`, `IPEL`이 모두 결과 매핑에 정확히 1개씩 존재해야 한다.
2. 각 유형은 실제 `nextQuestion(history)` 흐름으로 생성한 20개 응답 이력에서 산출되어야 한다.
3. 각 유형은 원칙 고수 응답에서 `-S`, Phase 2 양보 응답에서 `-M`이 모두 산출되어야 한다.
4. 생성된 응답 이력은 `validateReachableHistory(history, { allowComplete: true })`를 통과해야 한다.
5. 산출된 유형은 `buildResultViewModel()`을 통해 제목, 설명, 캐릭터 파일명까지 조립되어야 한다.

현재 자동 검증:

- `tests/algorithm/type-coverage.test.ts`
- 16개 유형 `S` 도달성 16건
- 16개 유형 `M` 도달성 16건
- 결과 매핑 완전성 1건
- 축별 목표 문자 우세 점수 확인 1건

## 4. 문항 수준 및 적절성 기준

자동 테스트에서 확인하는 기준:

| 항목 | 기준 |
|---|---|
| 인코딩 | 깨진 문자, 한자형 모지바케, replacement character가 없어야 함 |
| 문항 길이 | prompt 80자 이하, 선택지 90자 이하 |
| 선택지 구성 | A/B 선택지가 서로 달라야 함 |
| 표현 방식 | 실제 정당, 후보, 선거, 투표 등 직접 정치 행위 유도 표현 금지 |
| 낙인 방지 | 우월, 열등, 틀렸다, 위험한 사람 같은 단정 표현 금지 |
| 배점 일관성 | Phase 1과 balanced 문항은 A=왼쪽 축, B=오른쪽 축 |
| S/M 일관성 | Phase 2 단방향 분기는 A=principle, B=concession |
| 결과 카피 | 16개 결과 제목/부제/설명은 깨짐 없이 최소 길이 기준 충족 |

현재 자동 검증:

- `tests/algorithm/question-quality.test.ts`
- 문항/선택지 가독성 및 금칙 표현 확인
- 문항별 점수 방향 계약 확인
- 결과 매핑 카피 완전성 확인

## 5. 현재 판단

현재 로컬 MVP는 알고리즘 QA 관점에서 16개 유형과 S/M 태그가 모두 도달 가능한 상태입니다. 또한 사용자에게 직접 노출되는 문항과 결과 카피는 최소한의 인코딩, 길이, 금칙 표현, 배점 방향 검증을 통과합니다.

단, 이는 MVP 수준의 구조 및 카피 안전성 검증입니다. 정식 출시 전에는 다음 검증이 추가로 필요합니다.

- 20명 이상 내부 파일럿을 통한 문항 이해도 확인
- 특정 선택지가 항상 더 매력적으로 보이는지에 대한 응답 편향 검토
- 축별 문항 난이도와 추상도 균형 검토
- 결과 설명이 사용자를 특정 정치 집단으로 단정하지 않는지 재검수
- 실제 공유 이미지와 결과 일러스트 적용 후 화면 QA
