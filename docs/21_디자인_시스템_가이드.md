# 📄 21_디자인_시스템_가이드

## 1. 문서 개요

본 문서는 폴다임(Poldigm) 서비스의 시각적 일관성을 유지하기 위한 **UI/UX 디자인 시스템 명세서**입니다.

프론트엔드 스타일링은 **CSS 변수와 컴포넌트 scoped CSS**를 기본으로 합니다. 전역 디자인 토큰은 `app.css`의 `:root`에 정의하고, 컴포넌트별 세부 스타일은 각 Svelte 컴포넌트의 `<style>` 블록에서 관리합니다.

## 2. 디자인 핵심 원칙 (Design Principles)

기존의 딱딱하고 올드한 '정치 성향 테스트'의 느낌을 지우기 위해 다음 3가지 시각적 원칙을 고수합니다.

1. **Dark & Edgy (다크 모드 기본):**
    
    - 새벽에 혼자 몰입해서 푸는 '심리 게임'이나 '사이버펑크 세계관' 느낌을 주기 위해 다크 테마를 기본 베이스로 사용합니다. 밝은 형광 포인트 컬러로 텍스트와 UI에 엣지를 더합니다.
        
2. **Focus on Content (미니멀리즘):**
    
    - 복잡한 배경 패턴을 지양하고, 질문 텍스트와 선택지에만 시선이 꽂히도록 여백을 넓게 씁니다.
        
3. **Fluid Motion (부드러운 인터랙션):**
    
    - 버튼 터치, 화면 전환, 게이지 바 상승 등 모든 상태 변화에는 부드러운 트랜지션 애니메이션을 적용하여 딱딱한 설문지 느낌을 줄입니다.

## 3. CSS 변수 토큰

### 3.1. 전역 토큰 정의

아래 토큰은 `src/app.css` 또는 이에 준하는 전역 CSS 파일의 `:root`에 정의합니다.

```css
:root {
  color-scheme: dark;

  --color-bg: #111827;
  --color-page-bg: #000000;
  --color-surface: #1f2937;
  --color-surface-hover: #374151;
  --color-border: #374151;
  --color-text: #f9fafb;
  --color-text-muted: #9ca3af;
  --color-text-soft: #d1d5db;

  --color-primary: #6366f1;
  --color-accent: #22d3ee;
  --color-strong: #f43f5e;
  --color-moderate: #2dd4bf;

  --font-family-base: "Pretendard", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;

  --font-size-display: 30px;
  --font-size-question: 20px;
  --font-size-choice: 16px;
  --font-size-body: 14px;
  --font-size-caption: 12px;

  --line-height-tight: 1.2;
  --line-height-question: 1.375;
  --line-height-normal: 1.5;
  --line-height-relaxed: 1.625;

  --font-weight-normal: 400;
  --font-weight-medium: 500;
  --font-weight-bold: 700;
  --font-weight-black: 900;

  --radius-md: 12px;
  --radius-lg: 16px;

  --space-page-x: 20px;
  --space-card: 20px;
  --viewport-max-width: 448px;
  --choice-min-height: 120px;
  --cta-height: 56px;

  --transition-fast: 150ms ease;
  --transition-normal: 300ms ease-out;
  --shadow-primary-glow: 0 0 15px rgba(99, 102, 241, 0.5);
}
```

### 3.2. 토큰 사용 원칙

- 색상, 글자 크기, 주요 여백, radius, transition은 하드코딩하지 않고 CSS 변수를 우선 사용합니다.
- 컴포넌트 고유의 일회성 배치는 scoped CSS에서 직접 선언할 수 있습니다.
- 결과 이미지 캡처 영역은 CSS 변수 기반 스타일을 사용해 브라우저와 캡처 라이브러리 간 차이를 줄입니다.

## 4. 타이포그래피 (Typography)

가독성이 가장 중요한 서비스이므로, 국영문 혼용 시 밸런스가 좋은 `Pretendard`를 단일 기본 폰트로 사용합니다.

| 용도 | CSS 변수 | 크기 | 굵기 | 줄간격 |
|---|---|---|---|---|
| Display (결과 유형 타이틀) | `--font-size-display` | 30px | `--font-weight-black` | `--line-height-tight` |
| Main Question (질문 본문) | `--font-size-question` | 20px | `--font-weight-bold` | `--line-height-question` |
| Choice Button (선택지 텍스트) | `--font-size-choice` | 16px | `--font-weight-medium` | `--line-height-normal` |
| Body (결과지 상세 설명) | `--font-size-body` | 14px | `--font-weight-normal` | `--line-height-relaxed` |
| Caption (부가 정보, 퍼센트) | `--font-size-caption` | 12px | `--font-weight-medium` | `1` |

질문 본문은 단어가 흩어져 보이지 않도록 `letter-spacing: -0.01em`까지 허용합니다. 일반 UI 텍스트에는 기본 자간을 사용합니다.

## 5. 핵심 UI 컴포넌트 규격 (Components)

### 5.1. 앱 셸 (App Shell)

```css
.app-shell {
  min-height: 100dvh;
  background: var(--color-page-bg);
  color: var(--color-text);
  font-family: var(--font-family-base);
}

.mobile-viewport {
  width: 100%;
  max-width: var(--viewport-max-width);
  min-height: 100dvh;
  margin: 0 auto;
  background: var(--color-bg);
  padding-inline: var(--space-page-x);
}
```

### 5.2. 진행 상태 바 (Progress Bar)

- 위치: 화면 최상단 `sticky`.
- 두께: 6px.
- 배경: `--color-surface`.
- 채움: `--color-primary`.
- 애니메이션: `width var(--transition-normal)`.

```css
.progress-track {
  height: 6px;
  background: var(--color-surface);
}

.progress-value {
  height: 100%;
  background: var(--color-primary);
  transition: width var(--transition-normal);
}
```

### 5.3. 양자택일 선택지 카드 (Choice Card)

질문 화면의 가장 중요한 인터랙션 요소입니다. 엄지손가락으로 쉽게 탭할 수 있도록 큼직하게 설계합니다.

```css
.choice-card {
  width: 100%;
  min-height: var(--choice-min-height);
  padding: var(--space-card);
  border: 1px solid transparent;
  border-radius: var(--radius-lg);
  background: var(--color-surface);
  color: var(--color-text-soft);
  font-size: var(--font-size-choice);
  font-weight: var(--font-weight-medium);
  line-height: var(--line-height-normal);
  transition:
    transform var(--transition-fast),
    border-color var(--transition-fast),
    background-color var(--transition-fast),
    color var(--transition-fast);
}

.choice-card:hover,
.choice-card:focus-visible {
  border-color: var(--color-primary);
  background: var(--color-surface-hover);
  color: var(--color-text);
}

.choice-card:active {
  transform: scale(0.98);
}
```

### 5.4. 하단 고정 액션 버튼 (Floating CTA)

```css
.floating-cta {
  position: fixed;
  left: 16px;
  right: 16px;
  bottom: calc(24px + env(safe-area-inset-bottom));
  height: var(--cta-height);
  border: 0;
  border-radius: var(--radius-md);
  background: var(--color-primary);
  color: var(--color-text);
  font-weight: var(--font-weight-bold);
  box-shadow: var(--shadow-primary-glow);
}
```

## 6. 레이아웃 및 여백 (Layout & Spacing)

- PC 환경에서도 모바일 앱을 쓰는 경험을 제공하기 위해 콘텐츠 최대 너비는 `--viewport-max-width`인 448px로 제한합니다.
- 모바일 좌우 여백은 `--space-page-x`인 20px을 기본으로 사용합니다.
- 질문 텍스트와 선택지 버튼 영역 사이에는 최소 40px 이상의 여백을 둡니다.
- 하단 고정 버튼이 콘텐츠를 가리지 않도록 주요 스크롤 영역에는 `padding-bottom: calc(var(--cta-height) + 48px + env(safe-area-inset-bottom))`를 적용합니다.

## 7. 구현 가이드

- 전역 CSS에는 reset, 폰트 선언, CSS 변수, 앱 배경만 둡니다.
- 화면별 레이아웃과 컴포넌트 스타일은 Svelte 컴포넌트의 scoped CSS에 둡니다.
- 반복 사용되는 버튼, 카드, 게이지는 Svelte 컴포넌트로 분리하되, 스타일 토큰은 CSS 변수로만 참조합니다.
- Tailwind CSS는 기본 스택에 포함하지 않습니다. 필요 시 프로토타입 용도로만 별도 브랜치에서 검토합니다.
