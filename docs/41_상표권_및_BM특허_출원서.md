# 📄 41_상표권_및_BM특허_출원서

## 1. 문서 개요

본 문서는 폴다임(Poldigm) 서비스의 독창적인 브랜드 가치를 보호하고, 핵심 경쟁력인 '적응형 분기 알고리즘(CAT)'에 대한 카피캣(모방 서비스) 난립을 법적으로 방어하기 위해 작성된 **지적재산권(IP) 확보용 사전 명세서**입니다. 본 문서는 외부 특허법인(변리사) 미팅 시 핵심 설명 자료로 활용됩니다.

본 문서는 출원 가능성이나 등록 가능성을 보장하지 않는 내부 초안입니다. 실제 상표/특허 출원 전에는 변리사를 통해 선행상표, 선행기술, 청구항 범위, 지정상품/서비스업 적정성을 검토해야 합니다.

## 2. 제1부: 상표권 출원 명세서 (Trademark)

서비스의 이름과 관련된 독점적 권리를 확보하기 위해 런칭 전 선출원을 진행합니다.

### 2.1. 출원 상표 정보

- **상표명 (국문/영문 혼용):** 폴다임 / Poldigm
    
- **상표의 형태:** 문자 상표 (Word Mark) 및 로고타입(추후 디자인팀 에셋 추가 시 복합상표로 출원)
    
- **네이밍의 식별력:** 'Political/Policy'와 'Paradigm'의 결합으로 만들어진 조어(Coined Word)이므로 상표 등록 가능성 및 식별력이 매우 높음.
    

### 2.2. 지정 상품 및 서비스업 분류 (NICE 분류 기준)

상표권은 등록한 카테고리 내에서만 보호받으므로, 향후 서비스 확장(커뮤니티, 앱 출시 등)을 고려하여 넓게 지정합니다.

- **제 09 류 (소프트웨어 및 앱):** 다운로드 가능한 소프트웨어, 성향 분석용 컴퓨터 소프트웨어, 모바일 애플리케이션.
    
- **제 41 류 (엔터테인먼트 및 제공업):** 온라인 심리/성향 테스트 제공업, 인터넷을 통한 오락정보 제공업.
    
- **제 42 류 (IT 서비스업):** 서비스형 소프트웨어(SaaS), 웹사이트 디자인 및 호스팅업 (클라우드 기반 진단 서비스).
    
- **제 45 류 (사회/개인 서비스업):** 온라인 소셜네트워킹 서비스업, 개인의 성향 및 심리 분석 서비스.
    

## 3. 제2부: BM 특허 출원 명세서 (Business Method Patent)

단순한 심리테스트 '아이디어' 자체는 특허의 대상이 되지 않으므로, '응답 데이터에 따라 실시간으로 문항이 분기되고 가중치가 연산되는 시스템적 처리 과정'을 정보통신기술(ICT)과 결합하여 BM 특허로 청구합니다.

### 3.1. 발명의 명칭 (가칭)

**"극한 딜레마 가중치 기반의 적응형 사회적·정치적 가치관 분석 방법 및 이를 수행하는 서버 시스템"**

_(Method and System for Adaptive Social and Political Tendency Diagnosis Based on Extreme Dilemma Weighting)_

### 3.2. 발명의 배경 및 해결하려는 과제

- **기존 기술의 문제점:** 기존의 5점 척도(매우 동의~매우 비동의) 기반 테스트는 사용자가 중립적인 답변('보통')을 선택할 수 있어 뚜렷한 성향을 도출하기 어렵고, 고정된 설문지가 노출되어 몰입도가 저하됨.
    
- **해결 과제:** 중립 선택지를 배제한 강제 양자택일 인터페이스를 제공하고, 사용자의 이전 응답 데이터를 서버가 실시간으로 분석하여 그 신념의 한계를 시험하는 '심층 꼬리 문항(Phase 2)'을 동적으로 서빙함으로써, 혼합 응답 사용자와 원칙 고수형 사용자의 차이를 기술적으로 구분하고자 함.
    

### 3.3. 특허 청구항의 뼈대 (핵심 로직 Flow)

변리사가 실제 청구항을 작성할 때 기준이 되는 시스템 동작 순서도입니다.

1. **[제 1단계: 탐색 데이터 수집]** 클라이언트 단말기가 서버로부터 'Phase 1(탐색기)' 양자택일 딜레마 문항 데이터를 수신하여 화면에 출력하고, 사용자의 제1 응답 데이터를 수집하는 단계.
    
2. **[제 2단계: 예비 성향 연산]** 서버가 상기 제1 응답 데이터를 수신하여, 기 설정된 4개의 기준 축(C/I, T/P, M/E, O/L)에 대한 예비 점수(방향성)를 연산하는 단계.
    
3. **[제 3단계: 적응형 문항 동적 매칭]** 서버가 상기 예비 점수가 특정 임계값(예: 40점)에 도달한 축을 식별하고, 해당 성향의 한계 상황을 가정한 'Phase 2(심층기)' 문항을 서버 전용 문항 모듈에서 추출하여 클라이언트 단말기로 전송하는 단계.
    
4. **[제 4단계: 심층 데이터 수집 및 검증]** 클라이언트 단말기가 사용자의 제2 응답 데이터를 수집하여 서버로 전송하고, 서버는 봇 검증 모듈(Turnstile)을 통해 정상적인 입력인지 판별하는 단계.
    
5. **[제 5단계: 최종 태그 연산 및 렌더링]** 서버가 제1 및 제2 응답 데이터의 가중치를 합산하여 16가지 성향 코드 중 하나를 결정하고, Phase 2 심층 문항에서 최종 승리 지표의 원칙을 고수한 횟수에 따라 강도 태그(-S 또는 -M)를 부여하여 결과 데이터를 반환하는 단계.
    

### 3.4. 본 발명의 기대 효과 (기술적 진보성)

- 동적 문항 서빙을 통해 클라이언트 측의 메모리 부담을 최소화하고, 전체 문항 풀을 브라우저에 내려주지 않음.
    
- 클라이언트 브라우저에 가중치 연산 로직과 전체 문항 데이터를 노출하지 않는 엣지 서버(Edge Server) 기반의 보안 구조를 통해 알고리즘 탈취를 방지함.
    

## 4. 변리사 미팅 전 준비(첨부) 자료 리스트

특허법인에 본 명세서 초안을 전달할 때, 기술에 대한 이해를 돕기 위해 개발 및 기획팀에서 아래 자료를 취합하여 함께 제출해야 합니다.

- `[첨부 1]` **시스템 아키텍처 다이어그램:** SvelteKit 엣지 서버와 Supabase 간의 데이터 통신 구조도 (`30_시스템_아키텍처_설계서` 참조).
    
- `[첨부 2]` **플로우차트 (Flowchart):** 사용자가 랜딩 페이지에 접속하여 결과를 공유하기까지의 전체 UI/UX 흐름도.
    
- `[첨부 3]` **Phase 1 → Phase 2 분기 로직 예시표:** 특정 응답(예: 공공안전 선택)이 어떻게 꼬리 문항(예: 영장 없는 사찰)으로 이어지는지 보여주는 트리 구조 엑셀 시트.
    
- `[첨부 4]` **실제 구동 화면 캡처본 (또는 와이어프레임):** 카드형 양자택일 UI와 결과 도출 화면 스크린샷 (`20_UIUX_화면설계서_Wireframe` 참조).

- `[첨부 5]` **선행 서비스/선행기술 조사 메모:** 기존 정치 성향 테스트, MBTI형 테스트, CAT(Computerized Adaptive Testing) 기반 문항 서빙 서비스와의 차이점 정리.

- `[첨부 6]` **서버 구현 증거:** 문항 풀이 클라이언트에 노출되지 않고 SvelteKit 서버 API에서만 분기/점수 계산되는 코드 구조 또는 로컬 MVP 시연 자료.

변리사 검토 전 확인할 질문:

| 질문 | 확인 목적 |
|---|---|
| `Poldigm` 문자 상표의 선행 유사 상표가 있는가 | 상표 등록 가능성 |
| NICE 분류 09/41/42/45 전체가 필요한가 | 출원 비용과 보호 범위 조정 |
| BM 특허로 보호 가능한 기술적 구성은 무엇인가 | 단순 아이디어와 기술 구현의 분리 |
| CAT/심리테스트 선행기술 대비 차별점은 무엇인가 | 신규성/진보성 검토 |
| 문항/결과 카피는 저작권 보호 대상으로 별도 관리할 수 있는가 | 콘텐츠 보호 전략 |
