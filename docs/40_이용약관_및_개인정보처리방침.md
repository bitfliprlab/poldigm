# 📄 40_이용약관_및_개인정보처리방침

## 1. 문서 개요

본 문서는 폴다임(Poldigm) 서비스 운영 시 발생할 수 있는 법적 분쟁을 예방하고, 사용자 데이터 수집에 대한 투명성을 확보하기 위한 **이용약관(ToS) 및 개인정보처리방침(Privacy Policy)** 초안입니다.

폴다임은 '무인증(비로그인) 서비스'이지만, 접속 기기 종류, 접속 국가 코드, 테스트 결과 데이터를 수집하므로 이를 명확히 고지해야 합니다. 이 내용은 웹사이트 하단(Footer)에 링크 형태로 제공되어야 합니다.

본 문서는 서비스 기획/개발 단계의 초안이며, 실제 배포 전에는 운영 주체, 사업자 정보, 문의처, 관할 법령, 광고/분석 도구 사용 여부를 기준으로 법률 전문가 검토를 받는 것을 권장합니다.

## 2. 제1부: 서비스 이용약관 (Terms of Service)

**제1조 (목적)**

본 약관은 폴다임(이하 "회사")이 제공하는 성향 진단 테스트 서비스(이하 "서비스")의 이용과 관련하여 회사와 사용자 간의 권리, 의무 및 책임 사항을 규정함을 목적으로 합니다.

**제2조 (서비스의 제공 및 변경)**

1. 서비스는 회원가입 없이 누구나 무료로 이용할 수 있습니다.
    
2. 회사는 운영상, 기술상의 필요에 따라 제공하고 있는 서비스의 전부 또는 일부를 사전 공지 없이 변경하거나 종료할 수 있습니다.
    

**제3조 (사용자의 의무 및 어뷰징 금지)**

1. 사용자는 서비스를 이용할 때 다음 각 호의 행위를 하여서는 안 됩니다.
    
    - 매크로 프로그램, 봇(Bot) 등을 이용하여 비정상적인 트래픽을 유발하거나 서버에 부하를 주는 행위
        
    - 서비스의 적응형 분기 알고리즘 및 데이터를 리버스 엔지니어링(Reverse Engineering)하거나 무단으로 추출하는 행위
        
2. 회사는 비정상적인 접근(예: Cloudflare Turnstile 검증 실패, 비정상적인 플레이 타임 등)이 감지될 경우 해당 요청을 차단하고 결과를 저장하지 않을 권리를 가집니다.
    

**제4조 (지적재산권 및 저작권 보호)**

1. 서비스 내에 포함된 후보 딜레마 문항 풀, 알고리즘 로직, 16개 유형별 결과 텍스트, 캐릭터 일러스트, UI/UX 디자인 등에 대한 저작권 및 지적재산권은 전적으로 '회사'에 귀속됩니다.
    
2. 사용자는 결과를 SNS에 공유하는 목적 외에, 회사의 사전 승낙 없이 콘텐츠를 영리 목적으로 복제, 무단 도용, 2차 가공하여 배포할 수 없습니다.
    

**제5조 (면책 조항)**

본 서비스가 제공하는 성향 진단 결과는 사용자에게 즐거움과 참고용 인사이트를 제공하기 위한 콘텐츠일 뿐이며, 어떠한 의학적, 심리학적, 법적 효력이나 전문적 진단을 대체하지 않습니다. 회사는 사용자가 테스트 결과를 바탕으로 한 행동이나 분쟁에 대해 책임을 지지 않습니다.

## 3. 제2부: 개인정보처리방침 (Privacy Policy)

**💡 핵심 요약: 폴다임은 이름, 전화번호, 이메일 등 '개인을 특정할 수 있는 식별 정보(PII)'를 수집하지 않습니다. 닉네임은 선택 입력할 수 있으나 서버에 저장하지 않으며, IP 주소 원문도 DB에 저장하지 않습니다.**

**제1조 (수집하는 데이터의 항목 및 수집 방법)**

회사는 서비스의 원활한 제공과 통계 분석을 위해 사용자가 테스트를 완료하는 시점에 다음과 같은 '비식별 데이터'를 자동으로 수집합니다.

1. **서비스 이용 기록:** 최종 도출된 성향 코드(예: CTMO-S), 각 4가지 축의 누적 득점(score), 테스트 시작부터 완료까지의 소요 시간(초).
    
2. **접속 환경 정보:** 접속 기기 종류(Mobile/PC/Tablet), 유입 경로(UTM Source), 접속 국가 코드(예: `KR`, `US`).
    
3. **선택 닉네임:** 사용자는 결과 화면 및 공유 이미지 개인화를 위해 닉네임을 선택 입력할 수 있습니다. 닉네임은 사용자의 브라우저에서만 처리되며 서버 DB에 저장되지 않습니다.
    
4. **수집 방법:** 사용자가 마지막 문항을 선택하고 결과 산출을 요청하는 순간 API를 통해 수집. 접속 국가 코드는 Cloudflare 요청 메타데이터를 바탕으로 서버에서 파생합니다.
    

**제2조 (데이터의 수집 및 이용 목적)**

수집된 비식별 데이터는 다음의 목적으로만 활용됩니다.

1. 사용자의 테스트 결과 산출 및 결과 화면 제공.
    
2. 전체 사용자의 사회적·정치적 가치관 분포 등 익명화된 통계 분석 및 트렌드 리포트 작성.
    
3. 매크로 및 봇(Bot)에 의한 어뷰징 행위 탐지 및 방어 (보안 목적).
    
4. 접속 국가 코드 기반의 국가 단위 이용 통계 분석. IP 주소 원문, 도시, 상세 위치는 저장하지 않습니다.
    

**제3조 (데이터의 보유 및 이용 기간)**

1. 수집된 데이터는 개인을 특정할 수 없는 익명 정보이므로, 서비스의 통계 분석을 위해 서비스 종료 시까지 영구적으로 보관 및 활용될 수 있습니다.
    
2. 사용자가 본인의 데이터 삭제를 요청하더라도, 데이터가 익명화되어 있어 특정 개인의 데이터를 식별해 낼 수 없으므로 개별 삭제 처리는 불가능합니다.
    

**제4조 (쿠키 및 로컬 스토리지의 이용)**

회사는 사용자에게 끊김 없는 경험을 제공하기 위해 브라우저의 '세션 스토리지(Session Storage)'를 사용합니다.

- **목적:** 테스트 진행 중 실수로 창을 닫거나 새로고침했을 때 이전 응답 내역을 유지하기 위함입니다.
    
- **선택 닉네임:** 사용자가 입력한 닉네임은 결과 화면 및 공유 이미지 개인화를 위해 브라우저 세션 상태에서만 사용할 수 있습니다.
    
- **보관 기간:** 해당 데이터는 브라우저 탭을 닫거나 테스트가 완전히 종료되면 사용자의 기기에서 자동 소멸되며, 서버로 전송되어 영구 저장되지 않습니다.
    

**제5조 (보안 조치 및 서드파티 서비스)**

회사는 악의적인 트래픽을 방어하기 위해 **Cloudflare Turnstile** 서비스를 사용하며, 데이터 베이스 관리를 위해 **Supabase** 환경을 이용합니다. 이 과정에서 봇 판별을 위한 필수적인 네트워크 정보가 해당 서드파티 제공업체에 의해 처리될 수 있습니다.

**제6조 (광고 및 분석 도구 적용 시 추가 고지)**

로컬 MVP와 비공개 알파 단계에서는 광고 스크립트를 사용하지 않는 것을 기본으로 합니다. 실제 MVP에서 Google AdSense, Google Analytics 또는 유사한 광고/분석 도구를 적용하는 경우 아래 내용을 개인정보처리방침과 UI에 추가 검토합니다.

1. 광고 또는 분석 도구가 쿠키, 광고 식별자, 브라우저 식별 정보, 방문 로그를 사용할 수 있는지 확인합니다.

2. 결과 유형, 응답 이력, 선택 닉네임을 광고 타겟팅 이벤트나 개인화 광고 세그먼트로 전송하지 않습니다.

3. 광고/분석 도구 제공업체, 처리 목적, 보관 기간, 거부 방법 또는 동의 철회 방법을 명시합니다.

4. 법령 또는 플랫폼 정책상 동의 배너가 필요한 경우, 테스트 시작 전에 과도한 이탈을 만들지 않는 범위에서 동의 UI를 제공합니다.

5. 광고 도입 시에도 테스트 진행 화면과 선택지 근처에는 광고를 배치하지 않습니다.

## 5. 현재 구현 반영

현재 로컬 MVP는 `/terms`, `/privacy` 최소 정적 페이지를 구현했습니다. 또한 실제 광고/분석 도구, Supabase 저장, Cloudflare Turnstile을 사용하지 않습니다.

현재 처리:

| 항목 | 로컬 MVP 상태 |
|---|---|
| 회원가입/로그인 | 없음 |
| 닉네임 서버 저장 | 없음 |
| 결과 저장 | 서버 메모리 mock 저장소 |
| 광고/분석 쿠키 | 없음 |
| IP 주소 원문 저장 | 없음 |
| 국가 코드 저장 | 로컬 MVP 없음. 실제 MVP에서는 Cloudflare 메타데이터에서 파생한 국가 코드만 통계용 저장 |
| `/terms`, `/privacy` | 최소 정적 페이지 제공 |
| 결과 공유 | 링크 복사/이미지 저장 중심 |

공개 MVP 전환 전 `/terms`, `/privacy` 페이지의 실제 수집 항목, 운영 주체, 문의처, 광고/분석 도구 사용 여부를 실제 값으로 갱신해야 합니다.
