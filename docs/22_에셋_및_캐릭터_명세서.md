# 📄 22_에셋_및_캐릭터_명세서

## 1. 문서 개요

본 문서는 폴다임(Poldigm)의 결과 화면, 공유 이미지, 랜딩 비주얼에 사용될 **MVP 에셋 계약**을 정의합니다. 디자인팀이 실제 일러스트를 제작하기 전에도 프론트엔드 개발자가 파일명, 비율, 대체 이미지, 캡처 영역을 고정하여 구현을 시작할 수 있도록 최소 규격을 제공합니다.

## 2. 에셋 원칙

- **모바일 우선:** 모든 캐릭터와 공유 이미지는 모바일 결과 화면에서 먼저 검증합니다.
- **타입 코드 기반:** 캐릭터 파일명은 결과 코드의 4자리 기본 유형(`CTMO`, `IPEL` 등)을 기준으로 관리합니다.
- **S/M 태그 분리:** 강도 태그는 동일 캐릭터 위에 배지, 컬러, 텍스트 톤으로 표현하며 캐릭터 파일을 `-S`, `-M`으로 중복 제작하지 않습니다.
- **구현 우선 대체 가능:** 최종 일러스트가 없을 때는 타입 코드 이니셜, 단색 실루엣, 또는 임시 PNG를 같은 파일명으로 배치해 UI 개발을 막지 않습니다.

## 3. 캐릭터 일러스트 파일 규칙

| 항목 | 규격 |
|---|---|
| 저장 위치 | `static/assets/characters/` |
| 파일명 | `character_{type_code}.png` |
| 예시 | `character_CTMO.png`, `character_IPEL.png` |
| 권장 크기 | 1024 x 1024 px |
| 표시 비율 | 1:1 정사각형 |
| 배경 | 투명 배경 PNG 권장 |
| 최소 표시 크기 | 모바일 결과 화면 기준 가로 220px 이상 |
| 대체 텍스트 | `{title} 캐릭터 일러스트` |

## 5. 공유 이미지 및 썸네일 규격

| 용도 | 파일/렌더링 규격 | 비고 |
|---|---|---|
| 인스타그램 스토리 저장 이미지 | 1080 x 1920 px, 9:16 | 결과 캡처 영역에서 생성 |
| X/트위터 공유 이미지 | 1200 x 675 px, 16:9 | 별도 OG 이미지 또는 서버/클라이언트 캡처로 생성 |
| 카카오/일반 OG 이미지 | 1200 x 630 px | `og_default.png` 및 결과별 OG 확장 가능 |
| 랜딩 메인 비주얼 | 1024 x 1024 px 또는 WebP/GIF | 16개 캐릭터 실루엣 콜라주 가능 |

## 6. UI 연동 계약

- 결과 매핑의 `character_img` 값은 파일명만 저장합니다. 예: `character_CTMO.png`.
- 프론트엔드는 `static/assets/characters/{character_img}` 경로로 이미지를 참조합니다.
- 이미지 로딩 실패 시 타입 코드 이니셜을 담은 대체 박스를 표시합니다.
- `-S` 결과는 `Strong Tag` 컬러와 강한 대비를, `-M` 결과는 `Moderate Tag` 컬러와 부드러운 대비를 결과 UI에서 적용합니다.

## 7. MVP 완료 기준

- 16개 `character_{type_code}.png` 슬롯이 정의되어 있어야 합니다.
- 결과 화면 캡처 영역에서 캐릭터, 제목, 4축 게이지, 워터마크가 한 화면 안에 들어와야 합니다.
- 공유 이미지 생성 시 텍스트가 잘리지 않고 `Poldigm.com` 워터마크가 포함되어야 합니다.
