# 📄 22_에셋_및_캐릭터_명세서

## 1. 문서 개요

본 문서는 폴다임(Poldigm)의 결과 화면, 공유 이미지, 랜딩 비주얼에 사용될 **MVP 에셋 계약**을 정의합니다. 디자인팀이 실제 일러스트를 제작하기 전에도 프론트엔드 개발자가 파일명, 비율, 대체 이미지, 캡처 영역을 고정하여 구현을 시작할 수 있도록 최소 규격을 제공합니다.

## 2. 에셋 원칙

- **모바일 우선:** 모든 캐릭터와 공유 이미지는 모바일 결과 화면에서 먼저 검증합니다.
- **타입 코드 기반:** 캐릭터 파일명은 결과 코드의 4자리 기본 유형(`CTMO`, `IPEL` 등)을 기준으로 관리합니다.
- **S/M 태그 분리:** 강도 태그는 동일 캐릭터 위에 배지, 컬러, 텍스트 톤으로 표현하며 캐릭터 파일을 `-S`, `-M`으로 중복 제작하지 않습니다.
- **구현 우선 대체 가능:** 최종 일러스트가 없을 때는 타입 코드 이니셜, 단색 실루엣, 또는 임시 PNG를 같은 파일명으로 배치해 UI 개발을 막지 않습니다.

## 3. 캐릭터 일러스트 파일 규칙

| 항목 | 규격 |
|---|---|
| 저장 위치 | `static/assets/characters/` |
| 파일명 | `character_{type_code}.png` |
| 예시 | `character_CTMO.png`, `character_IPEL.png` |
| 권장 크기 | 1024 x 1024 px |
| 표시 비율 | 1:1 정사각형 |
| 배경 | 투명 배경 PNG 권장 |
| 최소 표시 크기 | 모바일 결과 화면 기준 가로 220px 이상 |
| 대체 텍스트 | `{title} 캐릭터 일러스트` |

## 4. 캐릭터 아트 스타일

결과 일러스트는 사용자의 정치 성향을 단정하는 이미지가 아니라, 결과 유형을 SNS에서 공유하고 싶게 만드는 **사회적·정치적 가치관 카드**로 표현합니다. 실제 정치인, 정당, 선거 캠페인처럼 보이지 않도록 상징과 색상 사용을 절제합니다.

### 4.1. 공통 스타일 방향

| 항목 | 기준 |
|---|---|
| 스타일명 | Neon Persona Card Illustration |
| 표현 방식 | 세미 플랫 벡터 기반 캐릭터 일러스트 |
| 분위기 | 다크, 세련됨, 심리 게임, 미래적이지만 과하게 SF적이지 않은 톤 |
| 형태 | 상반신 또는 전신 3/4 구도, 결과 카드 중앙 배치에 적합한 단일 캐릭터 |
| 선/면 처리 | 또렷한 실루엣, 단순한 면 분할, 과도한 질감과 세밀 묘사는 지양 |
| 색감 | 어두운 UI 위에서 보이는 보라, 시안, 로즈, 민트 계열 포인트 사용 |
| 배경 | 투명 PNG 기본. 배경 효과가 필요하면 UI에서 별도 처리 |
| 표정 | 공격적이거나 조롱하는 표정보다 자신감, 사려 깊음, 몰입감 중심 |
| 완성도 기준 | 모바일 결과 화면에서 작게 보여도 캐릭터 성격이 읽혀야 함 |

### 4.2. 금지 또는 주의 요소

- 실제 정치인, 특정 정당, 특정 국가, 선거 캠프를 연상시키는 외형과 구도는 사용하지 않습니다.
- 국기, 정당 로고, 투표용지, 유세차, 의회 배지처럼 현실 정치 상징성이 강한 소품은 피합니다.
- 좌/우 진영을 직접 암시하는 색상 대비, 손동작, 슬로건, 포스터식 문구를 사용하지 않습니다.
- 특정 성별, 직업, 계층, 세대, 지역을 조롱하거나 고정관념으로 보이게 만드는 표현을 피합니다.
- 군복, 제복, 방패, 깃발 등 권위나 운동성을 암시하는 요소는 추상화해서 사용하며 실제 단체 상징처럼 보이지 않게 합니다.
- 결과 유형을 선악, 우열, 정상/비정상처럼 보이게 만드는 명암 대비와 표정 연출은 피합니다.

### 4.3. S/M 태그 표현 방식

강도 태그는 캐릭터 파일을 `-S`, `-M`으로 분리하지 않고, 동일 캐릭터 위에 UI 스타일을 덧입혀 표현합니다.

| 태그 | 표현 기준 |
|---|---|
| `-S` Strong | 결과 카드의 배지, 게이지, 포인트 컬러 대비를 강하게 적용. 캐릭터 자체는 더 공격적으로 바꾸지 않음 |
| `-M` Moderate | 배지, 게이지, 포인트 컬러를 부드럽게 적용. 캐릭터 자체는 동일하게 유지 |

### 4.4. 제작 및 검수 기준

- 16개 캐릭터는 같은 그림체, 같은 명암 밀도, 같은 디테일 수준을 유지합니다.
- 각 캐릭터의 차이는 정치적 라벨이 아니라 자세, 소품, 색 포인트, 실루엣으로 구분합니다.
- 결과 캡처 영역에서 제목, 캐릭터, 한 줄 요약, 4축 게이지, 워터마크가 함께 들어가야 하므로 캐릭터는 세로로 과도하게 길지 않게 제작합니다.
- 투명 배경 PNG를 우선 제공하고, 최종 배포 전 파일 용량을 모바일 로딩에 적합하게 최적화합니다.
- 로컬 MVP에서는 같은 파일명 규칙의 placeholder를 사용할 수 있으나, placeholder도 최종 캐릭터와 동일한 1:1 비율을 유지해야 합니다.

## 6. 공유 이미지 및 썸네일 규격

| 용도 | 파일/렌더링 규격 | 비고 |
|---|---|---|
| 인스타그램 스토리 저장 이미지 | 1080 x 1920 px, 9:16 | 결과 캡처 영역에서 생성 |
| X/트위터 공유 이미지 | 1200 x 675 px, 16:9 | 별도 OG 이미지 또는 서버/클라이언트 캡처로 생성 |
| 카카오/일반 OG 이미지 | 1200 x 630 px | `og_default.png` 및 결과별 OG 확장 가능 |
| 랜딩 메인 비주얼 | 1024 x 1024 px 또는 WebP/GIF | 16개 캐릭터 실루엣 콜라주 가능 |

### 6.1. OG 기본 이미지 파일 계약

| 항목 | 규격 |
|---|---|
| 저장 위치 | `static/assets/og/` |
| 기본 파일명 | `og_default.png` |
| 권장 크기 | 1200 x 630 px |
| 용도 | 랜딩, 결과 페이지 기본 OG/Twitter Card 이미지 |
| 필수 포함 | `Poldigm` 로고 또는 워드마크, 테스트 핵심 카피, 과도하게 민감하지 않은 비주얼 |

로컬 MVP에서는 실제 이미지가 없어도 되지만, 실제 MVP 배포 전에는 `static/assets/og/og_default.png`를 반드시 준비합니다. 결과별 OG 이미지는 공유 성과를 확인한 뒤 후속 확장으로 검토합니다.

## 7. UI 연동 계약

- 결과 매핑의 `character_img` 값은 파일명만 저장합니다. 예: `character_CTMO.png`.
- 프론트엔드는 `static/assets/characters/{character_img}` 경로로 이미지를 참조합니다.
- 이미지 로딩 실패 시 타입 코드 이니셜을 담은 대체 박스를 표시합니다.
- `-S` 결과는 `Strong Tag` 컬러와 강한 대비를, `-M` 결과는 `Moderate Tag` 컬러와 부드러운 대비를 결과 UI에서 적용합니다.
- 캐릭터 이미지의 `alt` 텍스트는 `{title} 캐릭터 일러스트` 형식을 사용합니다.
- placeholder는 같은 타입 코드와 동일한 표시 비율을 유지해야 하며, 최종 이미지 교체 시 레이아웃이 바뀌지 않아야 합니다.
- 파일 용량 최적화는 PNG 투명 배경을 우선하되, 랜딩 비주얼처럼 투명이 필요 없는 경우 WebP를 사용할 수 있습니다.

## 8. MVP 완료 기준

- 16개 `character_{type_code}.png` 슬롯이 정의되어 있어야 합니다.
- 최종 일러스트가 없더라도 16개 타입의 placeholder 표시 기준이 구현되어 있어야 합니다.
- 결과 일러스트의 공통 스타일, 금지 요소, S/M 표현 방식이 제작 기준에 반영되어야 합니다.
- `static/assets/og/og_default.png`가 실제 MVP 배포 전 준비되어 있어야 합니다.
- 결과 화면 캡처 영역에서 캐릭터, 제목, 4축 게이지, 워터마크가 한 화면 안에 들어와야 합니다.
- 공유 이미지 생성 시 텍스트가 잘리지 않고 `Poldigm.com` 워터마크가 포함되어야 합니다.

## 9. 현재 구현 반영

현재 로컬 MVP에는 실제 캐릭터 PNG가 포함되어 있지 않습니다. 대신 `src/lib/components/result/CharacterImage.svelte`가 `typeCode`를 기반으로 인라인 SVG placeholder를 렌더링합니다.

| 항목 | 현재 상태 |
|---|---|
| 캐릭터 파일명 계약 | `character_{type_code}.png` 유지 |
| 실제 PNG | 미적용 |
| 로컬 결과 화면 | 타입 코드 기반 인라인 SVG 표시 |
| 교체 방식 | 실제 MVP에서 `static/assets/characters/character_{type_code}.png` 추가 후 동일 UI 계약 유지 |
| 검증 | Playwright 결과 화면 캡처 및 `test:e2e:chrome` 통과 |

실제 일러스트 제작 시에도 결과 카드의 1:1 표시 영역과 `character_img` 파일명 계약은 바꾸지 않습니다.
