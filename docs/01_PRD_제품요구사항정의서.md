# 📄 01_PRD: 제품 요구사항 정의서 (Product Requirements Document)

## 1. 프로젝트 개요 (Overview)

- **프로젝트명:** 폴다임 (Poldigm)
    
- **프로덕트 형태:** 모바일 웹 기반 성향 진단 플랫폼 (Mobile-first Web App)
    
- **핵심 요약:** 기존의 지루하고 평면적인 정치/사회 성향 테스트를 탈피하여, 적응형 분기 알고리즘(CAT)을 통해 개인의 가치관을 16가지 기본 유형과 강도 태그(S/M)로 입체적으로 분석해 주는 바이럴 테스트.
    

## 2. 대상 사용자 및 문제 정의 (Target & Problem)

### 2.1. 타겟 유저

- **Primary:** 정치·사회 문제에 관심은 있으나 극단적인 양당 체제나 기존의 낡은 이념 프레임에 소속감을 느끼지 못하는 2030 세대.
    
- **Secondary:** MBTI 등 심리테스트 결과를 SNS(인스타그램, X)에 공유하며 자아를 표현하기 즐기는 일반 대중.
    

### 2.2. 해결하고자 하는 문제 (Pain Points)

- 기존 테스트의 지루함: "매우 동의함 ~ 매우 비동의함" 식의 5점 척도 설문은 몰입도가 떨어짐.
    
- 애매한 결과: 중도층이나 스윙보터를 단순한 '회색지대'로 취급하여 결과에 대한 공감대가 낮음.
    
- 바이럴 요소 부재: 결과를 타인과 공유하거나 논의하고 싶게 만드는 네이밍 및 시각적 매력이 부족함.
    

## 3. 제품의 핵심 가치 (Core Values & USP)

1. **몰입형 딜레마 (Zero 'Neutral'):** '보통' 선택지를 없애고 가혹한 양자택일 딜레마를 강제하여 유저의 진짜 가치관을 도출.
    
2. **적응형 분기 알고리즘 (CAT):** 유저의 이전 대답에 따라 다음 질문의 뎁스(Depth)가 달라지는 꼬리 질문 방식을 통해 게임 같은 경험 제공.
    
3. **입체적 결과 세분화:** 4가지 기준 축(C/I, T/P, M/E, O/L) 조합에 더해, 강경파(-S)와 온건파(-M) 태그를 부여하여 스윙보터를 정확히 타겟팅.
    

## 4. 주요 기능 요구사항 (Functional Requirements)

### 4.1. 온보딩 및 진입 (Zero Friction)

- **무인증 시스템:** 회원가입이나 로그인 절차 없이 랜딩 페이지에서 [테스트 시작] 버튼 클릭 즉시 1번 문항으로 진입해야 함.
    
- **소셜 프루프 노출:** 랜딩 페이지에 '현재 참여자 수'를 동적으로 노출하여 신뢰도와 호기심 유발.
    
- **선택 닉네임:** 사용자는 결과 화면 개인화를 위해 닉네임을 선택 입력할 수 있으나, 미입력 시에도 테스트와 결과 확인이 가능해야 함.
    

### 4.2. 테스트 진행 (Test Flow)

- **상태 표시:** 상단 프로그래스 바를 통해 진행률을 직관적으로 제공.
    
- **문항 노출 로직 (Phase 1 → Phase 2):**
    
    - Phase 1 (탐색기): 8문항. 기본 성향 파악.
        
    - Phase 2 (심층기): 12~16문항. Phase 1 결과에 기반한 극단적 한계 상황 제시.
        
- **세션 유지:** 새로고침 시에도 이전 응답 기록이 날아가지 않도록 로컬 스토리지 또는 세션 스토리지 기반 상태 관리.
    

### 4.3. 결과 도출 및 공유 (Result & Viral)

- **결과 연산:** 서버 사이드에서 유저의 응답 데이터를 기반으로 가중치를 합산하여 최종 유형(예: `CTMO-S`) 도출.
    
- **결과지 구성:**
    
    - 유형별 직관적 타이틀 (예: 광장의 따뜻한 연대자) 및 전용 캐릭터 일러스트.
        
    - 4개 축에 대한 나의 득점 퍼센트(%) 게이지.
        
    - S(강경)/M(온건) 태그에 따른 성향 설명 텍스트.
        
    - 베스트/워스트 궁합 유형 매칭.
        
- **공유 기능:**
    
    - 인스타그램 스토리(9:16), X/트위터(16:9) 비율에 최적화된 결과 이미지 다운로드 기능 (html2canvas 등 활용).
        
    - 클립보드 링크 복사 버튼.
        

## 5. 비기능 요구사항 및 기술 스택 (Non-functional)

|**구분**|**요구사항 및 사용 기술**|
|---|---|
|**프론트엔드**|SvelteKit, CSS 변수 + 컴포넌트 scoped CSS (모바일 최적화 반응형 UI)|
|**백엔드/인프라**|Cloudflare Pages + Pages Functions/Workers (정적 호스팅 무료 기반, API는 사용량 기준 과금)|
|**데이터베이스**|Supabase (`test_results` 결과 통계 저장 전용, 클라이언트 직접 접근 금지)|
|**보안 및 알고리즘**|문항 풀(JSON/TS 모듈) 및 점수 연산 로직은 SvelteKit 서버 전용 영역에 은닉 (클라이언트 노출 차단)|
|**어뷰징/통계**|Cloudflare Turnstile 도입, IP 원문 미저장, 국가 코드는 통계용으로만 저장|

## 6. 마일스톤 및 일정 (Schedule)

총 8주 스프린트 기준으로 진행합니다.

- **Week 1-2 (기획/문서화):** PRD 확정, 100개 문항 풀 및 가중치 로직 설계 완료, 16개 유형 텍스트 작성.
    
- **Week 3-4 (디자인/법무):** UI/UX 와이어프레임 설계, 디자인 시스템 및 캐릭터 일러스트 제작, 상표권 출원.
    
- **Week 5-6 (개발):** SvelteKit 프론트엔드 퍼블리싱, 분기 로직 API 연동, Supabase DB 스키마 및 RLS 설정.
    
- **Week 7 (QA/테스트):** 사내 알파 테스트, 모바일 크로스 브라우징 테스트, 알고리즘 득점 밸런스 조정.
    
- **Week 8 (런칭/마케팅):** 상용 서버 배포(Cloudflare), 초기 시드 타겟 커뮤니티 배포 및 SNS 공유 트래픽 모니터링.
