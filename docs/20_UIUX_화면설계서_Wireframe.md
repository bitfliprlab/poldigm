# 📄 20_UIUX_화면설계서_Wireframe

## 1. 문서 개요

본 문서는 폴다임(Poldigm) 서비스의 모바일 웹 화면 설계도(Wireframe)입니다. 앱 설치 없이 SNS 인앱 브라우저에서 즉시 실행되는 특성을 고려하여 **모바일 반응형(Mobile-first)** 레이아웃을 기준으로 작성되었습니다. 디자이너와 프론트엔드 개발자는 본 명세의 화면 배치와 인터랙션 요소를 바탕으로 UI를 구현합니다.

## 2. 공통 UI/UX 원칙 (Global Rules)

- **Zero Friction (제로 마찰력):** 화면 전환 시 불필요한 팝업, 로그인, 권한 요구를 철저히 배제합니다.
    
- **One Screen, One Action:** 한 화면에는 오직 하나의 뚜렷한 행동(버튼 클릭)만 요구하여 인지 부하를 최소화합니다.
    
- **Micro-Interaction:** 선택지를 누를 때나 화면이 넘어갈 때 햅틱 피드백(지원 기기 한정)과 부드러운 스와이프/페이드 애니메이션을 적용하여 '게임'을 하는 듯한 조작감을 줍니다.
    

## 3. 화면별 레이아웃 및 기능 명세

### 3.1. 메인 랜딩 화면 (Landing Page)

유저가 링크를 클릭하고 최초로 도달하는 화면입니다. 3초 안에 호기심을 자극하고 이탈을 막아야 합니다.

- **[Top] 헤더 영역**
    
    - **Logo:** 좌측 상단 `Poldigm` 텍스트 로고 (미니멀하고 세련된 산세리프 폰트).
        
- **[Center] 메인 비주얼 및 카피**
    
    - **Main Copy:** "100개의 딜레마, 당신의 진짜 스탠스를 파헤치다." (가장 큰 폰트, Bold)
        
    - **Sub Copy:** "숨겨진 사회적 페르소나와 16가지 가치관 스펙트럼"
        
    - **Visual Asset:** 16개 캐릭터의 실루엣이 겹쳐지거나 돌아가는 형태의 은은한 GIF 또는 3D 에셋.
        
- **[Bottom] 액션 영역 (Floating CTA)**
    
    - **Social Proof (신뢰도 부여):** "🔥 현재 `24,512`명이 테스트에 참여했어요" (숫자는 카운트업 애니메이션 적용)
        
    - **CTA 버튼:** 화면 하단에 고정된 거대한 둥근 사각형 버튼. (Label: **테스트 시작하기**)
        
    - **버튼 액션:** 클릭 시 `SessionStorage` 초기화 후 즉시 `/test` 라우트로 페이드 아웃 전환.
        

### 3.2. 테스트 진행 화면 (Test Progress Page)

오직 질문과 선택지에만 집중할 수 있도록 여백(Whitespace)을 극대화한 화면입니다.

- **[Top] 진행 상태바 (Progress Indicator)**
    
    - 뒤로 가기 버튼 (`<`)
        
    - **상태 표시:** `Phase 1 (탐색기)` 등 현재 단계 표시.
        
    - **Progress Bar:** 얇고 세련된 막대그래프로 진행률(%) 표시. 화면을 가로지르며 부드럽게 차오르는 애니메이션. (예: 5/20)
        
- **[Center] 딜레마 질문 영역**
    
    - **질문 텍스트:** 화면 중앙에 크고 명확하게 배치. (예: "범죄 예방을 위해 도심 곳곳에 안면 인식 CCTV를 설치해야 하는가?")
        
    - 질문이 길어질 경우 가독성을 위해 자간과 행간을 넉넉히 확보.
        
- **[Bottom] 양자택일 카드 버튼 영역**
    
    - 상/하로 꽉 차게 배치된 2개의 거대한 카드형 버튼 (A 선택지 / B 선택지).
        
    - **버튼 디자인:** 중립적인 색상으로 시작하되, 터치(Hover) 시 해당 선택지의 테마 컬러가 살짝 번지는 효과.
        
    - **인터랙션:** 버튼 클릭 시 해당 카드가 살짝 눌리는(Scale down) 애니메이션 후, 다음 질문 카드가 우측에서 좌측으로 슬라이드인(Slide-in) 됨.
        

### 3.3. 로딩 및 분석 화면 (Loading & Interstitial)

20번째 마지막 질문을 선택한 직후, 서버(API)에서 결과를 연산하는 동안 노출되는 화면입니다. 텐션을 유발합니다.

- **[Center] 분석 애니메이션**
    
    - "당신의 가치관 데이터를 분석 중입니다..." 텍스트 노출.
        
    - 선택했던 키워드들(공공 안전, 능력, 평등, 자유 등)이 스크린에 빠르게 떠올랐다가 사라지는 타이포그래피 애니메이션.
        
    - 진행률 표기: `0% -> 100%` (약 2~3초간 지속하여 신뢰감과 기대감 부여)
        
- **[Action]**
    
    - 100% 도달 시 `/result/[id]` 페이지로 전환.
        

### 3.4. 최종 결과 화면 (Result Page - SNS 공유 최적화)

본 서비스의 핵심이자 바이럴의 시작점입니다. 이미지가 세로로 길게 이어지는 스크롤 형태입니다.

- **[Capture Area] 이미지 캡처/다운로드 대상 영역 (9:16 비율 권장)**
    
    - _이 영역은 유저가 '이미지 저장'을 눌렀을 때 렌더링되어 사진첩에 저장되는 영역입니다._
        
    - **결과 타이틀:** "나의 소셜 페르소나는" / **[광장의 따뜻한 연대자]** (가장 눈에 띄게 강조)
        
    - **캐릭터 이미지:** 해당 유형을 대표하는 매력적인 일러스트 (화면 중앙).
        
    - **한 줄 요약:** "아무도 소외되지 않는, 자유롭고 평등한 세상을 향해."
        
    - **스탯 게이지 (4 Axis):**
        
        - 양극단이 표시된 4개의 가로 막대그래프 (예: `[C 공동체] 80% ■■■■□□ [I 개인] 20%`)
            
        - S/M 태그에 따라 색상의 진하기나 불꽃 이펙트 차등 적용.
            
    - **워터마크:** 하단 조그맣게 `Poldigm.com` 로고 및 테스트 접속 QR코드 삽입.
        
- **[Scroll Area 1] 상세 해석 영역**
    
    - **강경파/온건파 상세 설명:** `desc_S` 또는 `desc_M` 텍스트 출력. 가독성을 위해 2~3문장 단위로 단락 구분.
        
- **[Scroll Area 2] 궁합 영역 (Chemistry)**
    
    - 좌우 반분된 카드 UI 레이아웃.
        
    - **환상의 파트너:** 해당 유형 일러스트 + 타이틀 (예: 자유로운 보헤미안 혁명가)
        
    - **파국의 앙숙:** 해당 유형 일러스트 + 타이틀 (예: 냉철한 시스템 설계자)
        
- **[Bottom Floating] 공유 및 액션 영역 (Sticky Bottom)**
    
    - 스크롤을 내려도 항상 하단에 고정되어 있는 액션 버튼 그룹.
        
    - `[인스타 스토리 공유]` 버튼 (가장 강조)
        
    - `[이미지 저장]` 버튼
        
    - `[링크 복사]` 아이콘 버튼
        
    - `[테스트 다시 하기]` 텍스트 버튼
        

## 4. 모바일 UI 디자인 체크리스트 (디자이너 & 개발자 참고용)

1. **가독성:** 모바일 야외 환경(햇빛 아래)에서도 잘 보이도록 텍스트와 배경의 명도 대비를 충분히 확보할 것.
    
2. **터치 영역:** 테스트 질문 선택지 버튼의 터치 영역을 최소 `세로 120px` 이상 확보하여, 한 손(엄지손가락)으로 쾌적하게 탭할 수 있도록 설계할 것.
    
3. **다크 모드 지원 (Optional):** Poldigm 특성상 진지하고 힙한 느낌을 주려면 기본 테마를 다크 모드(Dark Theme)로 설계하는 것이 시각적 임팩트가 큼 (형광색 포인트 컬러 활용).
    
4. **이미지 생성(html2canvas) 퀄리티:** 결과 화면을 이미지로 다운로드할 때 폰트가 깨지거나 CSS 레이아웃이 어긋나지 않는지 여러 모바일 디바이스에서 집중 QA 필수.
