# 11_결과지_텍스트_맵핑_문서

## 1. 문서 개요

본 문서는 폴다임(Poldigm) 로컬 MVP에서 최종 결과 코드(예: `CTMO-S`)를 결과 화면에 렌더링하기 위한 결과 매핑 계약을 정의합니다.

2026-06-21 기준 권위 소스는 서버 전용 모듈 `src/lib/server/data/result-mappings.ko-KR.ts`입니다. 프론트엔드는 전체 매핑 테이블을 직접 보유하지 않고, `/api/submit-result` 또는 `/result/[id]` 서버 load가 반환한 `resultViewModel`만 렌더링합니다.

## 2. 결과 데이터 계약

```ts
type ResultMapping = {
  locale: 'ko-KR';
  type_code: string;
  title: string;
  subtitle: string;
  character_img: string;
  desc_S: string;
  desc_M: string;
  chemistry_best: string;
  chemistry_worst: string;
};
```

결과 화면에서 사용하는 공개 ViewModel:

```ts
type ResultViewModel = {
  typeCode: string;
  intensityTag: 'S' | 'M';
  title: string;
  subtitle: string;
  characterImg: string;
  description: string;
  chemistryBest: string;
  chemistryWorst: string;
};
```

`description`은 `intensityTag === 'S'`이면 `desc_S`, `M`이면 `desc_M`을 사용합니다.

## 3. 현재 구현된 16개 유형

| type_code | title | subtitle | chemistry_best | chemistry_worst |
|---|---|---|---|---|
| `CTMO` | 공동체의 원칙 설계자 | 안정, 성과, 질서를 하나의 운영 원칙으로 묶습니다. | `ITEL` | `IPEL` |
| `CTML` | 절제된 자율 수호자 | 오래된 기준을 존중하되 선택의 공간을 남깁니다. | `ITMO` | `CPEL` |
| `CTEO` | 질서 있는 돌봄 관리자 | 전통의 안정감 위에서 격차를 줄이는 방식을 찾습니다. | `ITEL` | `IPML` |
| `CTEL` | 온건한 공동체 조율자 | 익숙한 규칙과 서로를 돌보는 마음을 함께 놓습니다. | `IPEL` | `IPMO` |
| `CPMO` | 공공 혁신 추진가 | 더 나은 사회를 강한 실행력으로 밀어붙입니다. | `IPML` | `ITEL` |

| `CPEO` | 제도 개혁 분석가 | 불공정한 구조를 질서 있게 고쳐 나갑니다. | `CTEL` | `ITMO` |
| `CPEL` | 광장의 평등 연대자 | 더 자유롭고 덜 배제되는 공동체를 상상합니다. | `CTML` | `CTMO` |
| `ITMO` | 개인의 책임 전략가 | 각자의 능력과 질서 있는 경쟁을 신뢰합니다. | `CPML` | `CPEO` |
| `ITML` | 고전적 자유 장인 | 작은 규칙, 큰 책임, 조용한 자율을 선호합니다. | `CTML` | `CPMO` |
| `ITEO` | 조용한 안정 중재자 | 개인의 존엄과 오래된 안정감을 함께 살핍니다. | `CTEO` | `CPML` |
| `ITEL` | 생활권 중심 자유주의자 | 가까운 관계와 작은 공동체 안의 자유를 믿습니다. | `CTEL` | `CPMO` |
| `IPMO` | 실험적 효율 개척자 | 새로운 방식과 빠른 성과로 판을 바꿉니다. | `CPMO` | `CTEL` |
| `IPML` | 자유 실험 메이커 | 각자의 역량으로 새로운 판을 직접 만듭니다. | `CPML` | `CTEO` |
| `IPEO` | 유연한 공정 설계자 | 자유로운 선택과 출발선 보정을 함께 설계합니다. | `CPEO` | `CTMO` |
| `IPEL` | 자유로운 평등 개척자 | 덜 억압하고 더 넓게 포용하는 사회를 향합니다. | `CPEL` | `CTMO` |

## 4. 캐릭터 이미지 계약

모든 유형은 아래 파일명 규칙을 따릅니다.

```text
character_{type_code}.png
```

예:

```text
character_CTMO.png
character_IPEL.png
```

로컬 MVP에는 실제 PNG가 없어도 됩니다. 현재 구현은 `CharacterImage.svelte`에서 `typeCode`를 기반으로 인라인 SVG 캐릭터를 렌더링합니다. 실제 MVP 전환 시 `static/assets/characters/character_{type_code}.png`를 추가하면 같은 계약으로 교체할 수 있습니다.

## 5. 공유 텍스트 템플릿

사용자가 결과를 공유할 때 권장하는 기본 문구입니다.

```text
100개 후보 딜레마 중, 당신에게 도착한 20문항

나의 소셜 페르소나는 [{title}]
{subtitle}

내 가치관 스펙트럼은 {intensityTag} 타입.
나와 잘 맞는 파트너는 {chemistryBest}, 자주 충돌할 수 있는 유형은 {chemistryWorst}.

Poldigm에서 나의 사회적 페르소나를 확인해보세요.
```

로컬 MVP의 링크 복사는 현재 로컬 URL을 기준으로 동작합니다. 실제 MVP에서는 `PUBLIC_APP_BASE_URL`을 실제 도메인으로 교체합니다.

## 6. 결과 카피 QA 기준

결과 카피는 SNS 공유, SEO 미리보기, 광고 플랫폼 심사에 노출될 수 있으므로 아래 기준을 지켜야 합니다.

| 항목 | 통과 기준 |
|---|---|
| 낙인 방지 | 사용자를 특정 정당, 진영, 계층, 위험 집단으로 단정하지 않는다. |
| 표현 톤 | “정치 성향 판정”보다 “사회적 가치관/의사결정 경향”으로 설명한다. |
| S/M 태그 | `S`는 원칙 고수, `M`은 상황 조정으로 설명한다. 극단주의/우월/열등 표현은 피한다. |
| 궁합 표현 | `chemistryWorst`는 재미 요소로만 쓰며 혐오/조롱/배제 표현을 쓰지 않는다. |
| 광고 심사 | 허위 선거 정보, 실제 후보/정당 지지 유도, 혐오·차별·공포 조장 표현이 없어야 한다. |
| 공유 문구 | 개인의 민감 성향을 과도하게 폭로하는 문장으로 보이지 않아야 한다. |

현재 구현에서는 `tests/algorithm/question-quality.test.ts`가 결과 매핑의 깨진 문자, 최소 길이, 파일명 계약, 금칙 표현을 자동 검증합니다.

## 7. 검증 상태

현재 구현은 `tests/algorithm/type-coverage.test.ts`에서 16개 유형 전체가 `S`와 `M` 양쪽으로 도달 가능한지 검증합니다.

통과 기준:

- 16개 `type_code`가 정확히 1개씩 존재한다.
- 실제 `nextQuestion(history)` 흐름으로 만든 20문항 이력에서 각 유형이 산출된다.
- `buildResultViewModel()`이 모든 유형의 제목, 설명, 궁합, 캐릭터 파일명을 조립한다.
- `npm run test`가 통과한다.
