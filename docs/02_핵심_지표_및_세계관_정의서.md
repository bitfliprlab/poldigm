# 02_핵심_지표_및_세계관_정의서

## 1. 문서 개요

본 문서는 폴다임(Poldigm)이 측정하는 4가지 기준 축과 강도 태그(S/M)의 의미를 정의합니다. 문항 출제자와 콘텐츠 작성자는 본 문서를 기준으로 질문의 딜레마, 결과지 텍스트, 공유 문구를 작성합니다.

Poldigm은 특정 정당, 진영, 후보, 선거 행동을 맞히는 검사가 아닙니다. 사용자가 생활 속 딜레마에서 어떤 가치를 먼저 놓는지 확인하는 **사회적·정치적 가치관 테스트**입니다.

## 2. 4대 핵심 지표

각 지표는 선과 악의 대립이 아니라, 제한된 상황에서 어떤 가치를 더 우선하고 어떤 비용을 감수하는지에 대한 차이입니다. 결과 카피는 사용자를 특정 집단으로 단정하지 않고 “이런 판단 기준을 더 자주 사용한다”는 톤으로 설명합니다.

### 2.1. 제 1축: 사회적 초점 C/I

공동의 안전과 개인의 권리가 충돌할 때 어느 쪽을 먼저 보호하는지 보는 축입니다.

| 코드 | 공식 이름 | 핵심 질문 | 문항 작성 기준 |
|---|---|---|---|
| `C` | 공동체 | 모두의 안전과 공동 기준을 위해 개인의 일부 불편을 감수할 수 있는가 | 공익, 연대, 공동 기준, 피해 예방을 중심으로 작성 |
| `I` | 개인 | 공익이 중요해도 개인의 동의와 통제권을 먼저 지켜야 하는가 | 사생활, 자기결정권, 권리 침해 경계를 중심으로 작성 |

주의:

- `C`를 권위주의나 집단주의로 단정하지 않습니다.
- `I`를 이기주의나 무책임으로 단정하지 않습니다.
- 실제 감시 정책, 방역 정책, 특정 사건을 사실 판단처럼 묻지 않고 가상의 가치 충돌로 구성합니다.

### 2.2. 제 2축: 변화 태도 T/P

검증된 기준과 새로운 변화가 충돌할 때 어느 쪽에 더 무게를 두는지 보는 축입니다.

| 코드 | 공식 이름 | 핵심 질문 | 문항 작성 기준 |
|---|---|---|---|
| `T` | 전통 | 오래 검증된 관습과 제도를 바꿀 때 얼마나 신중해야 하는가 | 안정감, 연속성, 공통 예절, 점진적 변화를 중심으로 작성 |
| `P` | 진보 | 기존 기준이 누군가를 배제한다면 얼마나 빠르게 바꿔야 하는가 | 다양성, 제도 개선, 새로운 생활 방식, 포용을 중심으로 작성 |

명칭 기준:

- 공식 축 이름은 `전통(T)`과 `진보(P)`를 사용합니다.
- `보수`는 현실 정치 진영으로 오해될 수 있어 공식 축 이름으로 쓰지 않습니다.
- `전통`은 “옛것이 항상 옳다”가 아니라 “검증된 기준을 쉽게 버리지 않는다”는 의미입니다.
- `진보`는 “새것이 항상 옳다”가 아니라 “배제와 불편을 줄이기 위해 기준을 바꿀 수 있다”는 의미입니다.

### 2.3. 제 3축: 분배 정의 M/E

한정된 자원과 보상을 어떤 기준으로 나눌지 보는 축입니다.

| 코드 | 공식 이름 | 핵심 질문 | 문항 작성 기준 |
|---|---|---|---|
| `M` | 성과 | 기여와 노력의 차이를 보상에 얼마나 반영해야 하는가 | 성과, 책임, 효율, 차등 보상을 중심으로 작성 |
| `E` | 평등 | 출발 조건의 차이를 줄이기 위해 얼마나 보정해야 하는가 | 기본 안정, 기회 보정, 격차 완화, 보호를 중심으로 작성 |

주의:

- `M`을 냉정함이나 엘리트주의로 낙인찍지 않습니다.
- `E`를 무조건적 결과 동일화로 단정하지 않습니다.
- 문항은 “성과 보상”과 “격차 완화”의 비용을 모두 보여주어야 합니다.

### 2.4. 제 4축: 통치 방식 O/L

사회 시스템이 위험을 줄이기 위해 얼마나 강한 규칙을 둘지 보는 축입니다.

| 코드 | 공식 이름 | 핵심 질문 | 문항 작성 기준 |
|---|---|---|---|
| `O` | 질서 | 피해를 줄이기 위해 사전 기준과 규제를 얼마나 강하게 둘 수 있는가 | 예측 가능성, 규칙, 예방, 공적 책임을 중심으로 작성 |
| `L` | 자유 | 위험 가능성이 있어도 선택과 실험의 공간을 얼마나 넓게 둘 것인가 | 자율, 책임, 실험, 과잉 규제 경계를 중심으로 작성 |

주의:

- `O`를 통제 욕구나 억압으로 단정하지 않습니다.
- `L`을 방임이나 무질서로 단정하지 않습니다.
- 같은 선택 안에도 장점과 비용이 함께 드러나야 합니다.

## 3. 강도 태그 S/M

4개 축 조합 뒤에는 `S` 또는 `M` 강도 태그가 붙습니다. 강도 태그는 사용자의 가치관이 “강하다/약하다”를 평가하는 점수가 아니라, Phase 2 딜레마에서 원칙을 얼마나 유지했는지 설명하는 보조 정보입니다.

| 태그 | 이름 | 의미 | 결과 카피 톤 |
|---|---|---|---|
| `S` | 원칙 고수형 | Phase 2의 한계 상황에서도 선택한 가치의 원칙을 여러 축에서 유지함 | 확고함, 일관성, 기준을 세우는 힘을 설명 |
| `M` | 상황 조정형 | Phase 2의 한계 상황에서 예외와 조정을 인정함 | 유연성, 현실 감각, 타협 가능성을 설명 |

주의:

- `S`를 극단주의자, 위험 인물처럼 쓰지 않습니다.
- `M`을 우유부단함이나 신념 부족처럼 쓰지 않습니다.
- 결과 설명은 “확정적 진단”이 아니라 “의사결정 경향”으로 표현합니다.

## 4. 문항 작성 기준

문항은 정치 주장처럼 보이지 않는 생활형 가치 딜레마로 작성합니다.

권장:

- “공동 안전과 사생활 보호가 충돌합니다.”
- “검증된 규칙과 새로운 생활 방식이 부딪힙니다.”
- “성과 보상과 기본 안정 사이에서 기준을 정해야 합니다.”
- “위험을 줄이는 규칙과 자유로운 시도가 충돌합니다.”

주의:

- 실제 정당, 후보, 선거, 투표 행동을 직접 묻지 않습니다.
- 실제 정책명이나 시사 사건을 찬반 투표처럼 묻지 않습니다.
- 특정 사회 집단, 지역, 성별, 종교, 직업군을 평가하지 않습니다.
- “옳다/틀리다”, “우월/열등”, “정상/비정상”처럼 사용자를 낙인찍는 표현을 쓰지 않습니다.

## 5. 결과 카피 기준

결과 카피는 SNS 공유, 광고 심사, 검색 미리보기에 노출될 수 있습니다. 아래 기준을 지킵니다.

| 항목 | 통과 기준 |
|---|---|
| 공식 포지셔닝 | “사회적·정치적 가치관”으로 설명 |
| 사용자 표현 | “이런 상황에서 이 가치를 우선하는 경향”으로 설명 |
| 금지 표현 | 특정 정치 진영, 정당, 후보, 위험 집단으로 단정 |
| S/M 태그 | 원칙 고수/상황 조정 정도로 설명 |
| 공유 문구 | 민감 성향을 과도하게 폭로하지 않음 |

## 6. 현재 구현 반영

현재 로컬 MVP의 권위 소스는 아래 파일입니다.

| 구분 | 구현 위치 |
|---|---|
| 축 코드와 공식 라벨 | `src/lib/shared/constants.ts` |
| 서버 전용 문항 풀 | `src/lib/server/data/questions.ko-KR.ts` |
| 결과 매핑 | `src/lib/server/data/result-mappings.ko-KR.ts` |
| 문항 품질 검증 | `tests/algorithm/question-quality.test.ts` |
| 문서 정합성 검증 | `tests/docs/documentation-consistency.test.ts` |

현재 구현 문항은 실제 정당/후보/선거 행동을 직접 묻지 않고, 공동체/개인, 전통/진보, 성과/평등, 질서/자유의 가치 충돌 상황으로 작성되어 있습니다.
